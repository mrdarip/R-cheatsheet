Glossary of Terms: Encoding Messages

This glossary provides technical definitions for terms related to linear block codes and group homomorphisms as used in message encoding.

Binary Group \( (\mathbb{Z}_2^n, \oplus) \)

The set of all binary strings of length \( n \). It forms a group under the operation of bitwise addition modulo 2 (XOR), denoted by \( \oplus \). In this group, the identity element is the string of all zeros, and every element is its own inverse.

See also: Homomorphism, Codeword.

Homomorphism

A function \( \phi: G \to H \) between two groups that preserves the group operation, such that \( \phi(a \cdot b) = \phi(a) \ast \phi(b) \). In coding theory, an encoding map is a homomorphism from a smaller binary group to a larger one.

See also: Encoding Map, Kernel.

Encoding Map

An injective (one-to-one) homomorphism \( \phi: \mathbb{Z}_2^k \to \mathbb{Z}_2^n \) that transforms a message of length \( k \) into a codeword of length \( n \), where \( n > k \). This process adds redundancy to allow for error detection and correction.

See also: Generator Matrix, Codeword.

Message

The original data string of length \( k \) bits that needs to be transmitted. It is an element of the domain \( \mathbb{Z}_2^k \) of the encoding map.

See also: Encoding Map.

Codeword

The resulting string of length \( n \) bits after a message has been processed by the encoding map. The set of all possible codewords is the image of the encoding map \( \text{Im}(\phi) \).

See also: Hamming Weight, Minimum Distance.

Generator Matrix (\( G \))

A matrix used to define a linear encoding map. If \( x \) is a message vector, the corresponding codeword \( c \) is calculated as \( c = xG \). The rows of this matrix form a basis for the code.

See also: Parity-Check Matrix.

Hamming Weight (\( \omega \))

The number of non-zero coordinates in a binary string. For a codeword \( x \), \( \omega(x) \) represents the number of '1's it contains.

See also: Hamming Distance.

Hamming Distance (\( d \))

The number of positions at which two binary strings \( x \) and \( y \) of the same length differ. It can be calculated as the Hamming weight of their XOR sum: \( d(x, y) = \omega(x \oplus y) \).

See also: Minimum Distance.

Minimum Distance (\( d_{min} \))

The smallest Hamming distance between any two distinct codewords in a code. This value determines the error-detecting and error-correcting capabilities of the code. A code can detect up to \( d_{min} - 1 \) errors and correct up to \( \lfloor (d_{min}-1)/2 \rfloor \) errors.

See also: Hamming Distance.

Parity-Check Matrix (\( H \))

A matrix used to verify if a received string is a valid codeword. A vector \( c \) is a codeword if and only if \( H \cdot c^T = \mathbf{0} \). The kernel of the linear map defined by \( H \) is exactly the set of all codewords.

See also: Syndrome, Kernel.

Syndrome

The vector \( S \) obtained by multiplying the parity-check matrix \( H \) by a received (potentially corrupted) vector \( r \): \( S = H \cdot r^T \). If the syndrome is zero, the received vector is a valid codeword.

See also: Decoding, Error Vector.

Error Vector (\( e \))

A binary string representing the errors introduced during transmission. If \( c \) is the sent codeword and \( r \) is the received string, then \( r = c \oplus e \). The syndrome of \( r \) is solely determined by the error vector: \( H \cdot r^T = H \cdot e^T \).

See also: Syndrome.

Decoding

The process of recovering the original message from the received vector. In "Syndrome Decoding," the syndrome is used to identify the most likely error vector, which is then XORed with the received vector to retrieve the codeword.

See also: Syndrome, Encoding Map.

Kernel

For a homomorphism \( \phi: G \to H \), the kernel is the set of all elements in \( G \) that map to the identity element in \( H \). For a valid encoding map, the kernel should only contain the zero vector to ensure the map is injective.

See also: Parity-Check Matrix.

Extra Easy Explained Section

If you found the math above a bit confusing, here are the same concepts explained simply!

Binary Group (The Sandbox)

Imagine a world where the only numbers are 0 and 1, and adding \( 1 + 1 \) gives you 0. This "sandbox" is where all our messages live.

Homomorphism (The Rule-Follower)

A special kind of translation that respects the rules of our sandbox. If you add two messages and then translate them, it’s the same as translating them first and then adding them together.

Encoding Map (The Bodyguard)

A way to take your short message and turn it into a longer one by adding extra "check" bits. These extra bits act like bodyguards that protect your information during a messy trip through a noisy wire.

Message (The Secret)

The original, short piece of information you want to send to a friend.

Codeword (The Disguise)

The version of your secret that has been "dressed up" with extra bits so it can survive errors. It's what actually gets sent.

Generator Matrix (The Blueprint)

The instruction manual or "stamping machine" that tells you exactly how to add those extra bits to your message to create a codeword.

Hamming Weight (The 'One' Count)

Just a fancy way of saying: "How many 1s are in this string?"

Hamming Distance (The Difference)

How many bits you have to flip to turn one string into another. If "101" and "110" are compared, they differ in two spots, so the distance is 2.

Minimum Distance (The Safety Gap)

The smallest possible difference between any two valid codewords. The bigger this gap, the harder it is for a few errors to accidentally turn one valid message into a different valid message.

Parity-Check Matrix (The Security Guard)

A special tool that looks at a received string and checks if it "fits the pattern." If it doesn't, the guard knows an error happened.

Syndrome (The Error Alarm)

A small code generated by the Security Guard. If the syndrome is all zeros, everything is fine. If it's not zero, it's an alarm that points toward where the error might be.

Error Vector (The Damage)

A map of which bits got flipped (changed from 0 to 1 or vice versa) while the message was traveling.

Decoding (The Recovery)

The process of looking at the damaged message, using the syndrome alarm to fix the errors, and getting your original secret back.

Kernel (The Zero-Makers)

The group of all strings that our "Security Guard" matrix thinks look like a perfect zero. For a good code, only the actual zero string should map to zero.
